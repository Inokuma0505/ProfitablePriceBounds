# 弾力性ベース追加実験の整理

## 目的

既存実験の設定は変えずに、価格範囲の決め方だけを追加で比較する。

- 需要生成、学習データ数、反復回数、評価指標は既存実験と同じ
- 追加したのは「価格範囲をどう作るか」というルールのみ
- 既存の `paper_*.json` に追加手法の結果を追記する形で実行する

## 前提

この実験では、各商品の需要は線形モデルで近似している。

- 予測需要: `q_m(p) = b_{m,0} + sum_j b_{m,j} p_j`
- 評価対象: 売上 `p_m q_m(p)` の合計

ここでの弾力性ベース手法は、厳密な理論最適化というよりも、
「価格感応度から探索範囲を簡単に決める」ためのヒューリスティクスとして導入している。

## 追加した2つの手法

### 1. Elasticity Center

実装名:

- `elaopt60`
- `elaopt55`
- ...
- `elaopt05`

考え方:

- 他商品の影響は無視し、各商品を1商品ずつ独立に見る
- 商品 `m` について、自己係数だけを使った1変数の売上
  `R_m(p_m) = p_m (b_{m,0} + b_{m,m} p_m)`
  を最大化する
- その最適価格を、その商品の中心価格とみなす
- その中心の周りで幅 `w` の価格範囲を作る

価格範囲:

- 商品 `m` の中心価格を `c_m` とすると
- 範囲は `[c_m - w(c_m - 0.5), c_m + w(1.1 - c_m)]`
- ここで `w` は `0` から `1` の間のスケール係数として扱う
- この形にすることで、中心価格が端に近い場合でも `[0.5, 1.1]` をはみ出さない

狙い:

- 商品ごとの価格感応度を使って、商品別に自然な中心価格を作る
- ただし実装は簡単に保つ

### 2. Elasticity Bias

実装名:

- `elabias60`
- `elabias55`
- ...
- `elabias05`

考え方:

- 他商品の影響は無視し、自己価格弾力性だけを見る
- 基準価格は既存の固定幅法とそろえて `0.8` を使う
- 商品 `m` の自己価格弾力性は
  `| b_{m,m} * 0.8 / (b_{m,0} + b_{m,m} * 0.8) |`
  で計算する
- 各 run の商品内平均弾力性と比較して、範囲を上側または下側に寄せる

価格範囲:

- 弾力性が平均以下の商品の範囲: `[1.1 - w, 1.1]`
- 弾力性が平均より大きい商品の範囲: `[0.5, 0.5 + w]`

解釈:

- 弾力性が小さい商品は値上げに比較的強いので、上側の価格帯を残す
- 弾力性が大きい商品は値上げに弱いので、下側の価格帯を残す

狙い:

- 弾力性の大小だけで、上寄せか下寄せかを決める
- 説明しやすい単純なルールにする

## 幅の決め方

追加実験の幅は、既存の range 系手法とそろえて以下を使う。

- `0.60`
- `0.55`
- `0.50`
- `0.45`
- `0.40`
- `0.35`
- `0.30`
- `0.25`
- `0.20`
- `0.15`
- `0.10`
- `0.05`

理由:

- 既存の `fixed` と同じ刻みにすることで、公平に比較できる
- 横軸の「平均価格範囲幅」を同じスケールで比較できる
- `0.60` は全体範囲 `[0.5, 1.1]` に一致する
- `0.05` まで狭めることで、かなり強い探索制約まで確認できる

## 既存実験との関係

この追加実験では、既存の実験の根本は動かしていない。

- データ生成は同じ
- 学習モデルは同じ
- 100回反復も同じ
- 比較対象の `quan` `boot` `ebpa` `fixed` もそのまま

変えたのは、追加で比較する価格範囲生成ルールのみである。

## JSONへの反映

追加手法の結果は既存の `paper_*.json` に追記される。

- M=5, N=1000 -> `paper_1000.json`
- M=5, N=300 -> `paper_300.json`
- M=10, N=1000 -> `paper_10_1000.json`
- M=10, N=300 -> `paper_10_300.json`

キー名は以下の通り。

- `elaopt60` から `elaopt05`
- `elabias60` から `elabias05`

## 報告用の短い説明

弾力性ベースの追加実験として、2種類の価格範囲設定法を比較した。
1つ目は、各商品を独立に見たときの1商品最適価格を中心に範囲を狭める方法である。
2つ目は、自己価格弾力性の大小に応じて、価格範囲を上側または下側に寄せる方法である。
どちらも既存の固定幅法と同じ幅系列 `0.60` から `0.05` を用い、既存実験と同一条件で比較した。

## 図に載せている弾力性手法の詳細

現在、図に載せている弾力性ベース手法は `Elasticity = elaopt` である。
`elabias` も実装しているが、図からは外している。

### 実装の流れ

1. 既存実験と同じデータを用いて、各商品の需要関数を線形回帰で推定する。
2. 商品 `m` ごとに、他商品の価格を無視した1変数売上
   `R_m(p_m) = p_m (b_{m,0} + b_{m,m} p_m)`
   を最大化する。
3. その解を商品 `m` の中心価格 `c_m` とする。
4. 中心価格 `c_m` を使って、各商品の価格範囲を
   `[c_m - w(c_m - 0.5), c_m + w(1.1 - c_m)]`
   と定める。
5. その商品別価格範囲の中で、改めて多商品同時の売上最大化を解く。

### この手法のポイント

- 中心価格を決める段階では、各商品を独立に扱う
- ただし最終的な価格最適化では、推定需要の全係数を使う
- したがって、価格範囲の作成だけを単純化し、最終最適化自体は既存実験と同じ枠組みで行っている

### 幅パラメータの解釈

実装上の `w` は、固定幅法のような「絶対幅」ではなく、中心から上下限までの距離を何割残すかを表す係数である。

- `w = 1` のとき、範囲はちょうど `[0.5, 1.1]`
- `w = 0` に近づくほど、範囲は中心価格 `c_m` に縮む

今回の実験では、既存手法との比較をそろえるため、以下の系列を用いている。

- `0.60`
- `0.55`
- `0.50`
- `0.45`
- `0.40`
- `0.35`
- `0.30`
- `0.25`
- `0.20`
- `0.15`
- `0.10`
- `0.05`

このとき、各商品の実際の価格範囲幅は
`w((c_m-0.5) + (1.1-c_m)) = 0.6w`
となる。

例えば:

- `elaopt60` の実際の幅は `0.36`
- `elaopt30` の実際の幅は `0.18`
- `elaopt05` の実際の幅は `0.03`

### 報告用の説明文

図に示した弾力性ベース手法では、まず各商品を独立に見たときの1商品最適価格を計算し、それを中心価格とした。
その後、価格範囲 `[0.5, 1.1]` をはみ出さないように、中心から上下方向へ比例的に範囲を縮小した。
最後に、その商品別価格範囲の中で、多商品同時の売上最大化を解いた。
したがって、この手法は、価格範囲の作り方だけを弾力性ベースに変えた追加実験である。
